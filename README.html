Overview

An application to abstract and visualize the edit procedure of common webpages. Combine the two words Page and Editor, I create the Peditor | 'pedit…ô |.

Build & Launch

Install dependencies: 

npm install

Launch: 

npm start

Test & Debug

Auto test: 

npm test

Debug: 

npm run-script debug

Tech

  • Frameworks: Linux, Enginx, CouchDB, Node.
  • Libraries: Express.js, Underscore.js, jQuery.
  • Languages: Coffeescript, Javascript, html, css.

Design

Agile.

Here I should introduce a new web app architecture that I created, MSV. The design pattern on the editor is Command Pattern.

Client System

  1. A visual editable grid editor. Easily split the page into containers.

  2. Common widgets:

    • slider
    • music list
    • custom

  3. Widget editor.

  4. History control, this will be implement in future.

  5. Save and load manager.

Protocol

Mainly a file format for saving the edit's states is enough. I call the format Page Document, file extension '.pdoc'. It saves the edit commands and page resources.

Server System

The CouchDB is good enough, it has already covered most the requirements with its build-in functions.
